i asked the team about this function function deployedAddress and @dean.axelar give us the best explain, hope this help all wardens: the question : and can i know how this function calculated the address please ? function deployedAddress(address sender, bytes32 salt) internal pure returns (address deployed) { address deployer = address(uint160(uint256(keccak256(abi.encodePacked(hex'ff', sender, salt, DEPLOYER_BYTECODE_HASH))))); deployed = address(uint160(uint256(keccak256(abi.encodePacked(hex'd6_94', deployer, hex'01'))))); } the answer: address deployer = address(uint160(uint256(keccak256(abi.encodePacked(hex'ff', sender, salt, DEPLOYER_BYTECODE_HASH))))) This line computes the address where the CreateDeployer contract would be deployed via a CREATE2 operation, based on the sender (which is the address of the contract that calls the deploy function), a salt, and the DEPLOYER_BYTECODE_HASH (which is the keccak256 hash of the CreateDeployer contract's bytecode). deployed = address(uint160(uint256(keccak256(abi.encodePacked(hex'd6_94', deployer, hex'01'))))) The second line takes the address of the CreateDeployer contract (calculated in the previous step), and uses it to compute another address, this time using a different prefix (hex'd6_94) and suffix (hex'01). This additional hash operation is used to compute the address of the contract that will be deployed using the CREATE opcode by the CreateDeployer contract. (edited)

Viktor_Cortess 07/20/2023 18:08
There is no info about express receiving, so I googled and found this. Is it true? GMP Express Privileged clients (protocols) can be allowed by Axelar to use the Express version of the messaging protocol. A standard message can take several minutes for the Axelar network to fully approve the message and pass it to the final destination. The express message will still go through the Axelar network, but GMP Express Service will lend any sent tokens to the destination address while approval is happening. Once the complete approval is done, tokens are paid back to the GMP Express service. This process can speed up the cross-chain transfer more than ten times. The crucial part of GMP Express is complete trust between the privileged protocol and Axelar. The privileged protocol MUST implement the logic for token returns when a standard message is received. Otherwise, the privileged protocol will receive 2x more tokens. See ExpressExecutable for more information.
